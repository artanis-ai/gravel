End-to-end in a few minutes. Everything below ships today on v0.5; nothing here is stubbed.

## 1. Install

```bash
cd your-existing-app
npx @artanis-ai/gravel init    # or `uvx artanis-gravel init` for Python projects
```

The wizard auto-adds the SDK to your project deps and writes the config + dashboard mount. For Docker / CI / polyglot setups, see the [direct install path](/install).

## 2. Open the dashboard

Start your app, navigate to `/admin/ai`. You'll see the login screen if you're using default-password mode, or the dashboard immediately if you've configured `getUser`.

Default password is in your `.env.local` as `GRAVEL_ADMIN_PASSWORD`. On `localhost` you're auto-admin (no password needed) so you can iterate without logging in every time.

## 3. See your prompts

The Prompts tab lists every prompt the wizard found in your repo. By default that's `*.md` / `*.txt` files in conventional directories (`prompts/`, `templates/`, `assistants/`), plus any embedded string literals (`SYSTEM_PROMPT = "..."`) the scanner picked up.

The scan runs at `gravel init` time and on every git commit (via the installed pre-commit hook). To rerun manually or include directories the default scan skipped:

```bash
npx gravel scan --deep    # or `uvx gravel scan --deep`
```

Deep scan runs locally on your machine using your own LLM key, your code never leaves your device.

## 4. Edit and submit

Click a prompt → edit inline → click **Submit changes**. Gravel opens a GitHub PR via the shared GitHub App.

First-time setup: the dashboard prompts you to install the [Gravel GitHub App](https://github.com/apps/gravel-bot) on the relevant repo. Once installed, every subsequent submit goes through that App's installation token — your developers never see Gravel asking for their personal OAuth.

For embedded prompts, the SDK applies edits in descending char-start order and rewrites your `.gravel/manifest.json` so subsequent prompts' line/char offsets stay valid through the merge. The whole PR (source files + manifest update) lands atomically.

## 5. Get a trace

Any `openai.chat.completions.create` (or Anthropic, LangChain, Vercel AI SDK, or generic `fetch`) call in your app shows up in the Traces tab in real time. Tracing auto-patches at SDK import time, so there's nothing to wire by hand.

The Datasets and Evals tabs are the next pillars in flight, separately documented under the [evals reference](/reference/evals) when you're ready to start running judges against captured traces.
