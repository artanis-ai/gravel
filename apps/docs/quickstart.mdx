This walks through end-to-end. **Some steps are not fully wired yet** during pre-v0 — they're flagged inline.

## 1. Install

```bash
cd your-existing-app
pnpm add @artanis-ai/gravel    # or `uv add artanis-gravel` for Python projects
pnpm gravel init               # or `uv run gravel init`
```

The SDK package ships a CLI wrapper that lazy-fetches the matching binary on first run. For Docker / CI / polyglot setups, see the [direct install path](/install).

## 2. Open the dashboard

Start your app, navigate to `/admin/ai`. You'll see the login screen if you're using default-password mode, or the dashboard immediately if you've configured `getUser`.

Default password is in your `.env.local` as `GRAVEL_ADMIN_PASSWORD`.

## 3. See your prompts

The Prompts tab lists every prompt the lib found in your repo. By default that's `*.md` / `*.txt` files in conventional directories (`prompts/`, `templates/`, `assistants/`).

To detect prompts embedded in code (`SYSTEM_PROMPT = "..."` etc.), run:

```bash
pnpm gravel scan --deep    # or `uv run gravel scan --deep`
```

<Warning>Deep scan isn't implemented yet (pre-v0). It will run locally on your machine using your own LLM key — code never leaves your device.</Warning>

## 4. Edit and submit

Click a prompt → edit it inline → click **Submit changes**. Gravel creates a GitHub PR via the shared App.

<Warning>The GitHub App PR flow lands alongside v0 build. For pre-v0 testing the dashboard works but submission is stubbed.</Warning>

## 5. Get a trace

Once tracing auto-patches land in v1, any `openai.chat.completions.create` (or Anthropic, Langchain, Vercel AI SDK) call in your app will appear in the Traces tab.

For now, the Traces / Datasets / Evals tabs render placeholder empty states pointing at the relevant version.
