Every prompt Gravel knows about is recorded in `.artanis/manifest.json` at your repo root. The file is **committed alongside your code**. A pre-commit hook keeps it in sync.

## Why a manifest

Prompts live where they live — in `.md` / `.txt` files, or as string literals in your code. Gravel never moves them. So we need a *map* of where every prompt is, what it currently says (hashed), and what its stable ID is for the dashboard.

## Format

```json
{
  "version": 1,
  "lastFullScanCommit": "abc123…",
  "lastFullScanAt": "2026-05-05T10:00:00Z",
  "prompts": [
    {
      "id": "p_5f3a7c8e",
      "type": "file",
      "path": "prompts/triage_assistant.md",
      "hash": "sha256:abc…"
    },
    {
      "id": "p_8d2b9e1f",
      "type": "embedded",
      "path": "src/agents/summarizer.ts",
      "lineStart": 42,
      "lineEnd": 58,
      "charStart": 1230,
      "charEnd": 1820,
      "varName": "SYSTEM_PROMPT",
      "hash": "sha256:def…"
    }
  ]
}
```

Stable `id` values survive renames. Hashes are normalized (line endings + trailing-whitespace insensitive) so cosmetic changes don't churn.

## Two scan modes

| Mode | When | Cost | Detects |
| --- | --- | --- | --- |
| **Fast scan** | Pre-commit hook | Zero LLM | Edits to known prompts; new `.md` / `.txt` files |
| **Deep scan** | `npx @artanis-ai/gravel scan --deep` | One LLM call per code file (cached) | Embedded prompts in code |

The deep scan runs **on your machine** using your own LLM key (or Claude Code / Codex if installed). Source code never leaves your device.

## Pre-commit hook

The wizard installs a polite-blocking pre-commit hook by default. It runs `npx @artanis-ai/gravel manifest --check`. If the manifest is out of date, the commit is rejected with an actionable message:

```
Gravel: Your prompt manifest is out of date.
Run:    npx @artanis-ai/gravel manifest --update
Then:   git add .artanis/manifest.json && git commit

(To bypass: git commit --no-verify)
```

Bypass any time with `--no-verify`. The next commit will catch the drift.
