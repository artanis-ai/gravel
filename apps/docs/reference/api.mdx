## TypeScript

```ts
// Define + activate config
import { defineConfig } from '@artanis-ai/gravel'
export const config = defineConfig({ /* ... */ })

// Boot tracing (OpenAI / Anthropic / LangChain / Vercel AI / globalThis.fetch)
import '@artanis-ai/gravel/auto'

// Mount the dashboard handler (Next App Router)
import { createGravelHandler } from '@artanis-ai/gravel/next'
const handler = createGravelHandler({ config })

// Next Pages Router
import { createGravelHandler } from '@artanis-ai/gravel/next-pages'

// Fastify (typed plugin)
import { gravelFastifyPlugin } from '@artanis-ai/gravel/fastify'
await fastify.register(gravelFastifyPlugin(config), { prefix: config.mountPath })

// Express / Hono / generic Node — all use the (req, res) bridge
import { gravelHandler } from '@artanis-ai/gravel/node'
app.use(config.mountPath, gravelHandler({ config }))
// Hono uses `app.mount(...)` against the same handler

// Tracing context
import { withGravelMetadata, withTracingDisabled, gravelContext } from '@artanis-ai/gravel'
```

## Python

```py
import artanis_gravel.auto  # boot tracing (OpenAI / Anthropic / LangChain + fetch)

# FastAPI
from artanis_gravel.fastapi import create_gravel_router
router = create_gravel_router(config=config)
app.include_router(router, prefix=config.mount_path)

# Django
from artanis_gravel.django import gravel_urls
urlpatterns = [path('admin/ai/', include(gravel_urls(config)))]

# Flask (requires the `[flask]` extra for the a2wsgi bridge)
from artanis_gravel.flask import mount_on_flask
mount_on_flask(app, config)

# Generic ASGI
from artanis_gravel.asgi import GravelAsgiApp
gravel_app = GravelAsgiApp(config=config)

# Generic WSGI
from artanis_gravel.asgi import gravel_wsgi_app
wsgi = gravel_wsgi_app(config)
```

Every Python integration delegates to the same `_handler.py` dispatcher,
so the routes below behave identically across FastAPI / Django / Flask /
raw ASGI / raw WSGI. The TS handler runs the same route table in
TypeScript (`packages/sdk-ts/src/handler/routes/`), one file per domain.

## Internal HTTP API

The dashboard talks to the lib via these routes (mounted under `mountPath`):

| Method | Path                                  | Purpose                                                  |
| ------ | ------------------------------------- | -------------------------------------------------------- |
| GET    | `/api/auth/me`                        | Current user (role, firstName, productName, mountPath)   |
| POST   | `/api/auth/login`                     | Default-password login. Accepts JSON or form-encoded.    |
| POST   | `/api/auth/logout`                    | Clears the session cookie, 303 to `/login`               |
| POST   | `/api/auth/view-as`                   | Admin-only: toggle a view-as-user cookie                 |
| GET    | `/api/version`                        | Admin-only: current SDK + npm/PyPI `latest` + `hasUpdate`|
| GET    | `/api/migrations/status`              | Admin-only: pending migration count + dialect            |
| GET    | `/api/prompts`                        | List manifest prompts with 280-char previews             |
| GET    | `/api/prompts/:id`                    | Single prompt (file content or sliced embedded range)    |
| POST   | `/api/prompts/submit`                 | Submit drafts as a single PR via the Gravel GitHub App   |
| GET    | `/api/samples`                        | List traced LLM calls (paginated + filterable)           |
| GET    | `/api/samples/:id`                    | One sample with full input/output/metadata + feedback    |
| POST   | `/api/samples/:id/feedback`           | Leave score / comment / correction                       |
| GET    | `/api/github/status`                  | Is the Gravel GitHub App installed for this project?     |
| GET    | `/api/github/install`                 | Returns a redirect URL to start the GH App install flow  |
| GET    | `/api/github/install/callback`        | Callback the CP redirects to; writes `.env.local`        |
| GET    | `/`, `/login`, `/<spa-route>`         | Serves the bundled React dashboard SPA shell             |
| GET    | `/_assets/:id`                        | Bundled Vite assets (content-type per extension)         |

The Datasets and Evals dashboard tabs are scaffolded as placeholder
routes; their backend tables and HTTP surface are explicitly out of scope
through v0.5.x.

This API ships in lockstep with the dashboard; the only stability
guarantee is within a single SDK release. Both SDK packages are at
`v0.5.x` today.
