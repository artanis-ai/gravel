---
title: CLI
description: "gravel — single binary for install, manifest, and diagnostics."
---

```bash
gravel init                Run the install wizard (detect framework, write mount, wire deps)
gravel manifest --check    Verify manifest is in sync (used by pre-commit hook)
gravel manifest --update   Regenerate manifest from working tree (fast file-scan)
gravel manifest --list     Print human-readable summary
gravel scan --deep         Deep scan for inline string-literal prompts; delegates to
                           your installed claude / codex / aider CLI and uses your
                           own LLM key — source never leaves your machine
gravel migrate             Apply DB migrations (idempotent bootstrap + Alembic/Drizzle)
gravel doctor              Diagnostic: DB, manifest, hook, tracing, SDK version
gravel help                Show help
```

## Init flags

| Flag                          | Effect                                        |
| ----------------------------- | --------------------------------------------- |
| `--yes`, `--non-interactive`  | Assume yes to every prompt                    |
| `--api-key <key>`             | Pre-bake project key into `.env.local`        |
| `--project <id>`              | Pre-bake project ID into `.env.local`         |
| `--mount-path <path>`         | Override default `/admin/ai`                  |
| `--prompts`, `--no-prompts`   | Toggle the prompts pillar                     |
| `--traces`, `--no-traces`     | Toggle the traces pillar                      |
| `--no-deep-scan`              | Skip the optional deep scan                   |
| `--no-test-trace`             | Skip end-to-end test trace                    |

The wizard detects a dev server already listening on the framework's
default port and warns to restart it before opening the dashboard, so
newly-written mount code is actually picked up.

## Python equivalent

The Python SDK ships the same `gravel` binary; `uvx artanis-gravel` is
the wrapper. Every flag above works identically:

```bash
uvx artanis-gravel init                 # same flags
uvx artanis-gravel manifest --check
uvx artanis-gravel migrate
```
