---
title: Auth
description: "getUser callback + default password mode."
---

Two **mutually exclusive** modes. Configure one. The other is unreachable.

## getUser callback

```ts
auth: {
  async getUser(req) {
    // req has: url, method, headers, cookies.get(name), raw (framework-native)
    const user = await myAuthLib.getCurrentUser(req)
    if (!user) return null      // unauthorized — host app's login takes over
    return {
      id: user.id,
      firstName: user.firstName,
      role: user.isAdmin ? 'admin' : 'user',
    }
  },
}
```

A `null` return means *not authed*. Gravel will not fall back to default-password — your app handles the redirect.

## Default password

```ts
auth: {
  defaultPassword: process.env.GRAVEL_ADMIN_PASSWORD!,
}
```

Active only when `getUser` is absent. The wizard generates a 32-character random password and writes to `.env.local`. Sessions are HMAC-signed cookies (30-day TTL).

This mode is acceptable for production with appropriate hardening (rate-limited, constant-time compare, etc.). For per-user attribution + SSO, configure `getUser`.

## Two roles

`'user'` and `'admin'`. Admins can configure GH connection, manage credits, view-as user. Users can browse + edit prompts, leave feedback, view eval results.

## "View as user"

Admins can toggle this to preview the user experience. Sets a `gravel_view_as=user` cookie that demotes their effective role until cleared.
